## 2. 教学准则（我承诺的）

1. **通俗优先**：每章开头先用大白话讲"为什么需要这个"，再讲原理，最后才看代码
2. **目标锚定**：每个概念都要回答"这对你的智能客服有什么用？"
3. **用户输出驱动**：不让你被动听讲，关键节点让你自己先试（如用 PEAS 分析一个场景）
4. **不急躁**：你不说"懂了，继续"，我就停下来答疑
5. **即时应用**：每章学完立刻在 smart-cs-agent 项目里落地
6. **双重记录**：每次对话结束更新 LEARNING_LOG.md + 章节笔记

## 3. 每章学习循环

```
① 通俗讲解（为什么 → 原理 → 和客服项目的关系）
② 代码实操（跑示例 → 修改 → 理解每一行）
③ 项目应用（知识用到 smart-cs-agent）
④ 记录沉淀（写笔记 → 更新日志）
⑤ 你确认懂了 → 进入下一章
```

## 4. 跨对话接续协议

**新对话开始时，我必须**：
1. 读取 `memory/METHODOLOGY.md`（本文件）
2. 读取 `memory/LEARNING_LOG.md` 确认进度
3. 读取当前章节笔记
4. 从上次停下的地方继续，不需要你重复解释背景

**你需要说的**：只需"继续学习第 X 章"即可。
