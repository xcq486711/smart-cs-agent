## 1. 核心概念（我的理解）

### Agent 是什么？
Agent = 一个有"感官→大脑→手脚"的自主实体。不是被动问答的 Chatbot，而是能自己感知环境、做决策、调用工具的程序。

四要素：
- **传感器**: 获取外部信息（用户输入、API 返回）
- **执行器**: 影响环境（调用函数、发消息、改数据）
- **环境**: 所在的世界（对话上下文 + 数据库 + API）
- **自主性（大脑）**: LLM 驱动的决策能力

### Agent Loop（核心循环）
```
感知 → 思考（规划 + 选工具）→ 行动（执行工具）→ 观察（读取结果）→ 回到感知
```
这是 Agent 的心脏。不是一次想完，而是一步一步试探、调整。

### 五类传统 Agent（进化路线）
1. 反射式（恒温器）: if→then，最快但是傻子
2. 基于模型（自动驾驶）: 有"世界模型"，看不到也能脑补
3. 基于目标（GPS 导航）: 知道去哪，会规划
4. 基于效用（股票 AI）: 不止达成目标，还要"最优"
5. 学习型（AlphaGo）: 不靠人教，自己学会的

### 神经符号主义 = 这本书最重要的概念
- **符号主义** = 图书管理员（规则精确但脆弱）
- **亚符号主义** = 婴儿学认猫（看多了就会，黑箱）
- **神经符号主义** = 人类大脑：系统 1（直觉/LLM）+ 系统 2（理性/规则）协同

**LLM Agent 就是神经符号主义的实践**：LLM 提供直觉和模糊推理，工具调用和规则提供精确执行。

### PEAS 模型 — 分析任何 Agent 任务的框架
- P (Performance): 怎么算做得好？
- E (Environment): 在什么世界里？
- A (Actuators): 能做什么？
- S (Sensors): 能感知什么？

### Workflow vs Agent
- Workflow = 流程图，路径写死，可靠但不能应变
- Agent = 有脑子的人，灵活但可能出错
- 最佳实践：两者结合，Workflow 管骨架，Agent 填血肉

## 2. 关键问答

| 问题 | 回答 |
|------|------|
| 代码里的 `parents[2]` 是什么意思？ | `Path(__file__).resolve().parents[2]` 从当前文件向上走 2 级目录到达项目根目录，用来找 .env 文件 |
| 为什么用 `re.search` 解析而不是 JSON？ | 因为 LLM 输出的是自然语言格式的文本（Thought: ... Action: ...），不是结构化的 JSON。正则解析是 Agent 框架的常见做法 |
| 为什么不直接用 OpenAI 的 Function Calling？ | 第 1 章的目的是让你理解底层机制。Function Calling 把这个解析过程"黑盒化"了——你会用它但不懂它怎么工作。第 7 章自研框架时会深入 |
| DeepSeek 和 OpenAI 什么关系？ | DeepSeek API 完全兼容 OpenAI SDK 接口格式，所以用同一套 `openai` 库就能调用 |

### 关键代码模式（之后会反复用到）
```python
# 1. 兼容 OpenAI 的客户端
client = OpenAI(api_key=..., base_url=...)

# 2. System Prompt 定义 Agent 角色 + 工具 + 输出格式
system_prompt = "你是... 可用工具: ... 输出格式: Thought+Action"

# 3. Agent Loop
for i in range(max_steps):
    output = llm.generate(full_history, system_prompt)
    action = parse_action(output)      # 正则解析
    if action == "Finish": break
    result = tools[action]()           # 执行工具
    history.append(f"Observation: {result}")  # 观察结果
```

## 4. 项目应用

### 已产出
- `docs/design.md` v0.1：用 PEAS 分析了医疗咨询和旅游规划两个领域
- 确定了 Workflow + Agent 混合架构
- 识别了环境特性（部分可观察、随机性、多 Agent、序贯动态）

### 对智能客服的启示
- Chatbot 做不了智能客服——因为环境是部分可观察的，需要 Agent 主动查、主动问
- 医疗场景尤其需要"系统 2"把关：LLM 可以分析症状，但关键建议必须有规则兜底
- 旅游场景是典型的 Agents 强项：动态查天气 → 推景点 → 调整 → 预订
