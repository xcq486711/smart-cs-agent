## 1. 三种范式

### ReAct（推理+行动循环）
- 核心：思考一步→行动一步→观察结果→再思考。**边想边做，根据外部反馈动态调整**
- 适用：需要外部工具反馈、步骤不确定的任务
- 不适用：需要全局最优解的任务（走一步看一步可能到死胡同）
- Prompt 本质：利用 Decoder-Only 自回归机制，Thought 的 token 成为 Action 的上下文

### Plan-and-Solve（先规划后执行）
- 核心：先规划全部步骤（Planner）→ 再逐步执行（Executor）。**想和做严格分离**
- 适用：步骤可预规划、结构化强的任务
- 不适用：需要动态调整、意外情况多的任务
- 关键设计：Planner 输出 Python 列表而非自然语言，保证可解析性

### Reflection（自我反思循环）
- 核心：执行→反思→优化。**不做新的事，把旧的事做得更好**
- 不是"根据外部反馈修改"（那是 ReAct），是"自己审视自己然后改进"
- 需要 Memory 模块存储"执行-反思-优化"轨迹
- 关键设计：反思 Prompt 需要具体审查维度（而非泛泛的"检查一下"）

## 2. 范式对比

| 维度 | ReAct | Plan-and-Solve | Reflection |
|---|---|---|---|
| 思考vs行动 | 交替进行 | 严格分离 | 行动→审视→改进 |
| 灵活性 | 高 | 低 | 中（只改自己的输出） |
| 全局最优 | 差 | 好 | 改进单次输出 |
| 外部依赖 | 工具/Observation | 不需要 | 不需要 |
| 记忆需求 | prompt_history | 历史步骤+结果 | 结构化 Memory |
